# 设计:首页 RBAC 角色过滤

> 日期:2026-07-18 | 状态:待评审 | 关联 PRD: 首页流程向导

## 1. 目标
首页流程向导 + 我的任务按当前用户**角色**显示对应阶段(业务/开发/审批各看不同)。

## 2. 角色(现有 auth/permission.go)
| 角色 | 常量 | 首页可见阶段 |
|---|---|---|
| 业务 | RoleBusiness | ①需求 |
| 开发 | RoleDev | ②认领③编码④测试⑤核对⑥登记 |
| 审批 | RoleGatekeeper | ⑦审批⑧上线 |
| 管理员 | RoleAdmin | 全部 |

## 3. 后端改动(my-tasks)
`GET /my-tasks` 增加 `roles` 字段(调 `auth.Store.Roles(user, psID)`):
```json
{
  "roles": ["dev"],
  "toClaim": [...], "myDev": [...], "toApprove": [...], "toRelease": [...]
}
```
requirement handler 注入 `authStore`。

## 4. 前端改动(page.tsx)
- 拿到 `roles` 后,过滤 FLOW 节点(只显示角色对应的)
- 过滤任务分组(业务只看 toClaim,审批只看 toApprove/toRelease)
- admin 看全部

## 5. 测试方案
新建 3 个测试用户(不同角色):
- `biz1`(business)→ POST /users + AddMember(role=business)
- `dev1`(dev)
- `gate1`(gatekeeper)
分别登录看首页,验证角色过滤。

## 6. 改动文件
- `requirement/handler.go`:NewHandler 加 authStore + MyTasks 返回 roles
- `cmd/server/main.go`:传 authStore
- `frontend/app/page.tsx`:按 roles 过滤节点+任务
- 测试:新建 3 用户(SSH curl)
