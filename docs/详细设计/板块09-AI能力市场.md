# 智源 ANP 平台 · 板块 9 · AI 能力市场（详细设计）

> **文档定位**：平台愿景"AI 嵌入各应用前端及底层"的落点。把平台 AI 能力封装为可复用的技能/SDK/API，让任意业务系统（内部应用/第三方系统）一行代码挂上 AI，共用同一套治理（板块4/5）与算力（板块8）。是 9 大板块能力对外的**统一窗口**。
> **技术基线**：业务后端 Go（Gin + sqlx + zap）管理技能注册/API Key/用量/治理；技能执行与能力编排由 **Python 侧 LangGraph** 承载，模型调用经 **LiteLLM 网关**（板块8），Go 经 **gRPC** 调用。前端 Copilot SDK 为 **TypeScript** 包（npm 分发）。持久化 PostgreSQL（schema: `capability`）+ Redis（会话缓存/限流）+ MinIO（技能资源包）。
> **配套**：软件开发架构设计、开发标准与规范、总体方案 V1.0。
> **多租户**：所有数据带 `project_space_id` 作为路由键。

## 1. 定位与价值

### 1.1 定位

AI 能力市场是平台的**能力复用中心**。它解决一个核心问题：企业有多个业务系统（ERP/OA/CRM/工单/财务…），每个都想用 AI，但各自接入成本高、治理失控、重复造轮子。能力市场把平台已验证的 AI 能力（需求解析、文档生成、数据问答、流程审批、报表分析…）封装为标准技能，通过 SDK/API 对外输出，统一治理、统一算力、统一审计。

⭐**嵌入三层架构**：前端 Copilot SDK ↔ 技能市场 ↔ Go 网关 → Python 智能层 + 算力（板块8）+ 治理（板块4/5）。

```
任意业务系统                AI 能力市场                    平台内核
(ERP/OA/CRM/工单)                                          
    │                         │                              │
    │ 1. 一行SDK嵌入           │                              │
    │ <CopilotWidget />       │                              │
    │ ──────────────────────▶ │                              │
    │                         │ 2. 技能匹配+编排              │
    │                         │ ──────────────────────────▶ │ Python Agent
    │                         │                              │ + 板块8算力
    │                         │                              │ + 板块4规则
    │                         │ 3. 统一鉴权/限流/计费/审计   │ + 板块5安全
    │                         │ ◀────────────────────────── │
    │ ◀── AI 结果/建议 ────── │                              │
    │                         │                              │
    │ 4. 技能灰度/下线/治理    │                              │
    │     统一在能力市场管控   │                              │
```

## 3. 功能模块

### 3.1 技能市场 Skill Store

- **技能目录**：可复用技能的分类目录（需求解析/文档生成/数据问答/流程审批/报表分析/代码助手…）。
- **技能详情**：输入/输出 schema、能力描述、绑定规则、版本、调用统计、评分。
- **技能安装**：业务系统通过 APIKey 或 SDK 引用技能。
- **版本管理**：技能支持多版本，灰度切换。
- **技能搜索**：按场景/标签/能力搜索匹配技能。

### 3.2 前端 Copilot SDK

⭐"一行代码给任意前端挂载 AI"：

- **对话助手组件**：`<CopilotChat />` — 挂载即用的 AI 对话窗口。
- **智能表单**：`<SmartForm />` — AI 辅助填表/校验/推荐。
- **智能搜索**：`<SmartSearch />` — 语义搜索 + 自然语言查询。
- **上下文感知**：SDK 自动提取宿主页面上下文（DOM/路由/用户操作），给 Agent 提供场景。
- **框架兼容**：支持 React/Vue/原生 HTML，npm 包分发。

```typescript
// 业务系统一行嵌入示例
import { CopilotWidget } from '@anp/copilot-sdk';

<CopilotWidget
  apiKey="sk_anp_xxx"
  projectSpaceId="ps_finance"
  skills={['data-qa', 'report-gen', 'approval-assist']}
  context={{ page: 'invoice-list', role: 'accountant' }}
/>
```

### 3.3 底层 API 网关

Go 侧统一网关，所有技能调用经此：

- **统一鉴权**：APIKey + 项目空间校验。
- **限流**：按 Key/技能/IP 多维度限流（Redis 令牌桶）。
- **计费**：调用计入板块8用量（Token/算力消耗归属调用方）。
- **审计**：每次调用记录 caller/skill/输入摘要/输出摘要/耗时/成本。
- **灰度路由**：技能灰度发布（按比例/按租户切换版本）。

### 3.4 能力治理

- **技能绑定规则**：每个技能声明数据访问范围/权限要求/合规约束（来自板块4/5）。
- **灰度发布**：新版本技能按比例放量。
- **可下线**：紧急下线（安全漏洞/合规问题）一键生效。
- **用量监控**：按技能/调用方/时间的用量看板。
- **质量评分**：调用方反馈 + 成功率 + 延迟综合评分。

### 3.5 领域 Agent 商店

- **领域 Agent**：面向特定业务域的复合 Agent（财务 Agent/人力 Agent/客服 Agent…）。
- **按需启用**：业务域选择启用，配置数据源/权限边界。
- **组合能力**：领域 Agent 由多个技能编排组合而成。
- **与板块2 研发协作**：领域 Agent 的开发走研发工作台流程。

## 5. AI Agent 设计

> 能力编排 Agent 和嵌入适配 Agent 在 **Python 侧**（agent-runtime 服务，LangGraph），Go 经 gRPC 调用。Agent 产出遵守五约束。

### 5.1 Agent 清单

| Agent | 职责 | 输入 | 输出 | 自治级 |
|---|---|---|---|---|
| 能力编排 Agent (OrchestrationAgent) | 组合多个技能为复合能力，编排执行流 | 用户意图+上下文+可用技能集 | 编排计划+执行结果 | L1 |
| 嵌入适配 Agent (EmbeddingAdapter) | 按宿主应用适配交互格式与上下文提取 | 宿主类型+页面上下文+用户操作 | 适配后的请求+渲染指令 | L1 |

### 5.2 能力编排 Agent 决策逻辑

```
[user_intent + context] ─▶ [classify_intent]
                                    │
                          ┌─────────┴──────────┐
                          ▼                    ▼
                  [single_skill?]        [multi_skill?]
                          │                    │
                          ▼                    ▼
                  [execute_direct]      [plan_orchestration]
                                               │
                                    ┌──────────┴──────────┐
                                    │ 例: 数据问答→        │
                                    │     报表生成→        │
                                    │     流程审批         │
                                    └──────────┬──────────┘
                                               ▼
                                       [execute_chain]
                                               │
                                               ▼
                                       [emit_result]
```

- **技能发现**：根据意图从技能市场匹配候选技能（向量检索 + 标签匹配）。
- **编排计划**：多技能时生成执行链（DAG），声明数据依赖。
- **失败处理**：技能执行失败时降级（换技能/换模型/返回部分结果）。
- 出站模型调用经板块8 LiteLLM 网关。

### 5.3 嵌入适配 Agent

```
[host_app + page_context + user_action]
                    │
                    ▼
           [identify_host_type]  ── ERP / OA / CRM / 工单 / 自定义
                    │
                    ▼
           [extract_semantic_context]  ── DOM结构 / 路由 / 数据实体 / 用户角色
                    │
                    ▼
           [adapt_request]  ── 构造标准化的技能调用请求
                    │
                    ▼
           [adapt_response] ── 输出格式适配(卡片/表单/图表/文本)
```

- 适配不同宿主的交互范式（列表页/详情页/表单页/看板页）。
- 上下文脱敏：敏感字段不传入 Agent（遵守板块5数据分级）。

### 5.4 gRPC IDL 概要（capability_agent.proto）

```protobuf
service CapabilityAgentService {
  // 能力编排（单技能 or 多技能组合）
  rpc OrchestrateCapability(OrchestrateRequest) returns (OrchestrateResponse);
  // 嵌入适配（上下文提取 + 请求构造 + 响应适配）
  rpc AdaptEmbedding(AdaptRequest) returns (AdaptResponse);
}

message OrchestrateRequest {
  string project_space_id = 1;
  string trace_id = 2;
  string user_intent = 3;
  string host_app = 4;             // ERP / OA / CRM ...
  map<string, string> context = 5; // 页面上下文
  repeated string skill_ids = 6;   // 候选技能(可选, Agent 也可自选)
  string api_key_id = 7;
}

message OrchestrateResponse {
  string plan_id = 1;
  repeated OrchestrationStep steps = 2;  // 编排步骤
  string result = 3;                     // 最终结果(JSON)
  string render_hint = 4;               // 渲染提示(card/form/chart/text)
  repeated string skills_used = 5;
  string model_version = 6;
  float confidence = 7;
}
```

### 5.5 透明度要求

每次编排记录：选了哪些技能、编排计划、各步执行结果、用了哪个模型、置信度——全部落 CapabilityUsage 可追溯。

## 6. 数据模型（Go struct / PostgreSQL）

> schema: `capability`。所有表带 `project_space_id`、审计字段。Go 侧用 sqlx，struct tag 做 JSON 序列化与校验。

### 6.1 Skill（技能）

```go
// Skill 可复用 AI 技能
type Skill struct {
    ID              string    `db:"id" json:"id"`
    ProjectSpaceID  string    `db:"project_space_id" json:"project_space_id"` // "*" 表示全局可用
    Code            string    `db:"code" json:"code" validate:"required"`     // 唯一编码 data-qa / report-gen
    Name            string    `db:"name" json:"name" validate:"required"`
    Description     string    `db:"description" json:"description"`
    Category        string    `db:"category" json:"category" validate:"oneof=requirement doc_gen data_qa approval report code assistant"`
    InputSchema     string    `db:"input_schema" json:"input_schema"`         // JSON Schema (输入定义)
    OutputSchema    string    `db:"output_schema" json:"output_schema"`       // JSON Schema (输出定义)
    RequiredSkills  []string  `db:"required_skills" json:"required_skills"`   // JSONB 依赖的子能力
    BoundRuleIDs    []string  `db:"bound_rule_ids" json:"bound_rule_ids"`     // JSONB 绑定规则(板块4)
    DataAccessScope string    `db:"data_access_scope" json:"data_access_scope"` // 数据访问范围声明
    Version         string    `db:"version" json:"version"`                   // 语义化版本
    Status          string    `db:"status" json:"status" validate:"oneof=draft pending_review active gray deprecated offline"`
    GrayPercentage  int       `db:"gray_percentage" json:"gray_percentage"`   // 灰度比例 0-100
    RiskLevel       string    `db:"risk_level" json:"risk_level" validate:"oneof=low medium high"`
    IsPublic        bool      `db:"is_public" json:"is_public"`               // 是否对外暴露
    IconURL         string    `db:"icon_url" json:"icon_url"`
    Tags            []string  `db:"tags" json:"tags"`                         // JSONB
    CreatedBy       string    `db:"created_by" json:"created_by"`
    CreatedAt       time.Time `db:"created_at" json:"created_at"`
    UpdatedAt       time.Time `db:"updated_at" json:"updated_at"`
}
```

### 6.2 CopilotSession（前端会话）

```go
// CopilotSession 前端 Copilot SDK 会话
type CopilotSession struct {
    ID              string    `db:"id" json:"id"`

    ContextSnapshot string    `db:"context_snapshot" json:"context_snapshot"` // JSONB 页面上下文快照
    SkillIDs        []string  `db:"skill_ids" json:"skill_ids"`              // 启用的技能集
    Messages        []Message `db:"messages" json:"messages"`                // JSONB 对话消息
    Status          string    `db:"status" json:"status" validate:"oneof=active idle closed"`
    CreatedAt       time.Time `db:"created_at" json:"created_at"`
    UpdatedAt       time.Time `db:"updated_at" json:"updated_at"`
}

type Message struct {
    Role       string                 `json:"role"`        // user / assistant / system
    Content    string                 `json:"content"`
    SkillUsed  string                 `json:"skill_used,omitempty"`
    RenderHint string                 `json:"render_hint,omitempty"` // card / form / chart / text
    Metadata   map[string]interface{} `json:"metadata,omitempty"`
    Timestamp  time.Time              `json:"timestamp"`
}
```

### 6.3 APIKey（调用凭证）

```go
// APIKey 外部系统调用凭证
type APIKey struct {
    ID              string    `db:"id" json:"id"`
    ProjectSpaceID  string    `db:"project_space_id" json:"project_space_id" validate:"required"`
    AppName         string    `db:"app_name" json:"app_name" validate:"required"`  // 调用方应用名
    KeyHash         string    `db:"key_hash" json:"key_hash"`                      // 哈希存储(不存明文)
    KeyPrefix       string    `db:"key_prefix" json:"key_prefix"`                  // 前缀(展示用 sk_anp_xxx...)
    AllowedSkills   []string  `db:"allowed_skills" json:"allowed_skills"`          // JSONB 授权技能列表
```

### 7.6 Gin 路由示例

```go
// internal/capability/api/router.go

func RegisterCapabilityRoutes(r *gin.Engine, h *CapabilityHandler, auth middleware.AuthMiddleware) {
    // 统一调用入口(独立 APIKey 鉴权)
    r.POST("/api/v1/capabilities/invoke", auth.RequireAPIKey(), h.InvokeCapability)
    r.POST("/api/v1/capabilities/invoke:stream", auth.RequireAPIKey(), h.InvokeCapabilityStream)
    r.GET("/api/v1/capabilities/skills", h.ListSkills)
    r.GET("/api/v1/capabilities/skills/:sid", h.GetSkill)

    // Copilot 会话(SDK 用 APIKey)
    r.POST("/api/v1/capabilities/sessions", auth.RequireAPIKey(), h.CreateSession)
    r.POST("/api/v1/capabilities/sessions/:sid/messages", auth.RequireAPIKey(), h.SendMessage)
    r.GET("/api/v1/capabilities/sessions/:sid", auth.RequireAPIKey(), h.GetSession)

    // 项目空间级管理
    v1 := r.Group("/api/v1/projects/:psid/capabilities")
    v1.Use(auth.Require(), auth.RequireTenant())
    {
        // 技能管理
        v1.POST("/skills", h.CreateSkill)
        v1.PUT("/skills/:sid", h.UpdateSkill)
        v1.POST("/skills/:sid:submit", h.SubmitSkill)        // 🚪 评审
        v1.POST("/skills/:sid:approve", h.ApproveSkill)       // 🚪 审批
        v1.POST("/skills/:sid:gray", h.AdjustGray)
        v1.POST("/skills/:sid:offline", h.OfflineSkill)       // 🚪 下线/熔断

        // APIKey
        v1.POST("/api-keys", h.CreateAPIKey)
        v1.GET("/api-keys", h.ListAPIKeys)
        v1.PUT("/api-keys/:kid", h.UpdateAPIKey)
        v1.POST("/api-keys/:kid:revoke", h.RevokeAPIKey)

        // 领域 Agent
        v1.GET("/domain-agents", h.ListDomainAgents)
        v1.POST("/domain-agents/:did:enable", h.EnableDomainAgent)
        v1.PUT("/domain-agents/:did:config", h.ConfigDomainAgent)

        // 看板
        v1.GET("/usage", h.GetUsageDashboard)
        v1.GET("/usage/by-skill", h.GetUsageBySkill)
        v1.GET("/usage/by-app", h.GetUsageByApp)
        v1.GET("/quality", h.GetQualityDashboard)
    }
}
```

## 8. 规则与约束

> 本板块**读**规则治理中心（板块4）和安全合规中心（板块5）的规则资产，不自己维护规则。

### 8.1 知识资产消费清单

| 资产类型 | 内容 | 用途 | 强制度 |
|---|---|---|---|
| Rule | 能力开放范围 | 技能可被哪些系统/角色调用 | 🟥 强制 |
| Rule | 数据访问权限 | 技能声明数据访问范围，调用时校验 | 🟥 强制 |
| Rule | 嵌入合规 | 对外嵌入的数据脱敏/审计要求 | 🟥 强制 |
| Process | 技能上架流程 | 评审+审批+灰度 | 🟥 强制 |
| Process | 技能下线/熔断流程 | 评估+通知+执行 | 🟥 强制 |
| Knowledge | 技能编排模式库 | 编排 Agent 参考最佳实践 | 🟧 应遵循 |

### 8.2 Agent 五约束落地

| 约束 | 能力市场场景落地 |
|---|---|
| 可校验 | 技能输入/输出有 JSON Schema，调用前后校验 |
| 可追溯 | 每次调用 CapabilityUsage 全量记录（caller/skill/tokens/cost/trace_id） |
| 可回滚 | 技能灰度/下线可即时回退版本 |
| 守边界 | 编排 Agent 只在授权技能集内编排；数据访问不超声明范围 |
| 宥权限 | APIKey 受限于 allowed_skills + rate_limit + quota；不得越权调用未授权技能 |

### 8.3 嵌入合规约束

```
🟥 对外暴露(第三方系统)的技能:
   - 数据脱敏: 敏感字段(来自板块5分级)不传入 Agent
   - 审计强化: 输入/输出摘要全量审计
   - 限流加固: 更严格的 rate_limit
   - 合规评审: 技能 is_public=true 需额外安全评审

🟧 内部系统嵌入:
   - 标准鉴权/限流/审计
   - 数据访问按 project_space_id 隔离
```

## 9. 人机闸门

### 9.1 审批清单

| 闸门 | 触发场景 | 约束 | 自治级 |
|---|---|---|---|
| 🚪新技能上架 | 技能 submit → 评审 → approve | 质量+安全+合规评审 | L2 |
| 🚪技能对外暴露 | is_public=true | 额外安全评审 | L2-L3 |
| 🚪技能下线/熔断 | 紧急安全/合规问题 | 评估+通知+执行 | L2 |
| 技能调用 | 日常调用已上架技能 | 自动 | L0-L1 |
| 能力编排 | 多技能组合 | 自动+知会 | L1 |
| 灰度调整 | 调整放量比例 | 管理员操作 | L2 |

### 9.2 闸门执行机制

- Go 网关在调用前校验技能状态（非 active 的技能不可调用）。
- 新技能必须经 submit→approve 流程才进入 `active` 或 `gray` 状态。
- is_public 技能额外校验安全评审状态。
- 紧急熔断：管理员一键 `offline`，Go 网关即时拦截该技能所有调用。
