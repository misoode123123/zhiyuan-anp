# 显示规范化 · 应用名替代随机 ID + 各步骤时间

> 日期：2026-07-20 ｜ 状态：**✅ 已落地并验证（后端全量测试绿 + 前端 build/test 绿）**
> 触发：dogfooding 反馈——各中心（审批/发布/首页/应用部署）显示任务时是一串随机 ID（`chg_6b473972`），看不出属于哪个应用；且各阶段列表缺创建/更新时间，进度不可读；应用名缺强制人工起名。
> 前序：[核对门禁 P0](2026-07-20-核对门禁P0修复-PRD.md) + [期 2 联动](2026-07-20-核对门禁期2联动-PRD.md) 已落地。

## 1. 目标

1. **各中心任务显示应用名**，不再裸露 `chg_xxx`/`req_xxx` 随机 ID。
2. **应用名必须人工起名**，后端拒空/拒 ID 风格（`app_`/`chg_` 前缀、纯数字、过短）。
3. **各步骤（需求/变更/发布）列表显示创建 + 更新时间**，进度可读。

## 2. 现状实证（grep 于 2026-07-20）

### 2.2 数据缺口

- `change_request.source_id` 可能是 **app_id**（交互编码登记）或 **requirement_id**（AI 编码派生）；`/changes` 接口（`change.Store.List`）只返回 `chgCols`，**不带 app 名**。
- 前端 approvals/release 只拉 `/changes`，无 apps 列表可 join → 只能显示 ID。
- `appdeploy.Create`（handler.go:725）用 `in.Name` 但**不校验非空/合法**；`createBody.name` 非 binding required。
- 部分列表（approvals/release 待发布）只显示 `created_at`，无 `reviewed_at`/`updated_at`。

## 3. 方案

### 3.1 后端：change/release 列表带应用名（核心）

- `change.Store` 的 `chgCols` + `List`/`Get` 改 LEFT JOIN 出 app 名（双路径：source_id 直接 = app_id，或经 requirement.application_id）：
  ```sql
  SELECT c.*, COALESCE(a.name,'') AS app_name
  FROM change_request c
  LEFT JOIN appdeploy_application a
    ON a.id = c.source_id
    OR a.id IN (SELECT application_id FROM requirement WHERE id = c.source_id)
  ```
- `ChangeRequest` 加 `AppName string \`json:"app_name" db:"app_name"\``。
- release 列表（`release` 包）同理带 app 名（release→change→source→app）。

### 3.2 后端：应用名强制人工起名

- `appdeploy.Create`：name 去 trim 后校验——非空、长度 ≥ 2、不含 `chg_/req_/app_/rel_` 前缀、非纯数字；非法 → 400「应用名需人工填写（≥2 字符，非 ID）」。
- `createBody.Name` 加 `binding:"required"`。

### 3.3 前端：各点 ID→应用名 + 时间

| 文件                      | 改动                                                                   |
| ------------------------- | ---------------------------------------------------------------------- |
| `approvals/page.tsx`      | `{c.id}`→`{c.app_name}`（保留短 id 作副标）；加 created_at/reviewed_at |
| `release/page.tsx`        | 待发布 `{c.id}`→`{c.app_name}`；发布历史加 created_at                  |
| `applications/page.tsx`   | 待上线变更摘要 fallback 用 app_name；变更/发布条目加时间               |
| `page.tsx`（首页）        | toApprove/toRelease fallback 用 app_name + 时间                        |
| `workspace-frame.tsx:404` | submit 提示保留 change_id（操作反馈，短）                              |

### 3.4 dev 页编码任务（第二批，可后续）

- `dev/page.tsx` source_id（requirement）→ 需求标题；需 codetask 列表接口带 title，本次先标注 TODO，不阻塞核心。
