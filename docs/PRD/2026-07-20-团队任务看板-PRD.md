# 团队任务看板 + 指派

> 日期：2026-07-20 ｜ 状态：**实施中**
> 上游：[8人团队工程化完善方案](../2026-07-18-8人团队工程化完善方案.md)（协作维度）
> 前序：[真实鉴权 P0](2026-07-20-真实鉴权-撤模拟回退-PRD.md) 已落地（8 人 8 账号可登录）
> 触发：8 人协作核心——现状 my-tasks 是个人视角 + 认领制，团队看不到全局进度、不能指派任务。

---

## 1. 现状（核实于 2026-07-20）

- 首页 my-tasks（`requirement/handler.MyTasks`）：toClaim / **myDev（仅自己）** / toApprove / toRelease。个人视角。
- `Assign`（`requirement/handler:194`）：用 X-User 头**自己认领**（互斥）。**无指派**（指定给某人）。
- 无团队全局视图：谁在做什么、负载、瓶颈不可见。

## 2. 目标

1. **团队看板**：全队任务 Kanban（待认领 / 开发中 / 待审批 / 待上线 / 已交付），每卡片显示 assignee + 应用 + 时间。
2. **指派**：待认领任务可指派给指定人（管理员/lead 操作，不只自己认领）。

## 3. 方案

### 3.1 后端 team-tasks 接口（全队视角）

- `GET /project-spaces/:id/team-tasks`：全队聚合（不限 user）
  - `toClaim`：assignee 空 + status=specified
  - `inDev`：assignee 非空 + status=developing（**全队**，非仅自己）
  - `toApprove`：change pending
  - `toRelease`：change approved
  - `delivered`：requirement delivered（近 20 条）
- 复用 MyTasks 逻辑，inDev 不限 user + 加 delivered + 每项带 app_name/assignee/created_at。

### 3.2 Assign 支持指派

- `POST /requirements/:rid/assign` body 加 `assignee`（可选）
  - 空 → 自己认领（X-User，现状）
  - 有 → 指派给 `assignee`（管理员/lead）
- 互斥不变（已被他人认领 → 409）。

### 3.3 前端 /team Kanban

- 新页 `/team`（侧边栏 + tabs 加入口）
- 5 列 Kanban：toClaim / inDev / toApprove / toRelease / delivered
- 卡片：标题 + 👤 assignee + 应用 + 📅 时间 + 优先级
- 待认领卡片：指派按钮（选成员）→ assign 带 assignee

## 4. 改动清单

- `internal/requirement/handler.go`：`TeamTasks` + `Assign` 加 assignee body
- `internal/requirement/service.go`：`Assign` 支持 assignee 参数
- `internal/requirement/handler_test.go`：TeamTasks + 指派用例
- `platform/frontend/app/team/page.tsx`（新）+ sidebar/tabs 加入口

## 5. 验收

1. `/team` 看板显示全队任务 5 列，每卡片有 assignee + 应用 + 时间。
2. 待认领任务可指派给某人（assignee 设定），被指派人在自己 my-tasks 看到。
3. 已被他人认领的任务指派/认领 → 409。
