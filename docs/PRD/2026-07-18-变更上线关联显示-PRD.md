# PRD: 变更-上线关联显示

> 日期: 2026-07-18 | 状态: 待评审

## 1. 问题
1. **首页待上线**: 只显示"变更 chg_xxx + status",看不到是哪个应用、什么功能、谁提交、什么时候
2. **应用部署上线**: 点「上线」只知道版本号+1,不知道上线的是哪个功能/变更

## 2. 目标
- **待上线列表**: 显示 功能摘要 + 应用名 + 提交人 + 提交时间
- **上线操作**: 上线前显示"本次将上线哪些 approved 变更",确认后才上线

## 3. 设计

### 3.1 首页待上线(改 my-tasks 返回 + 前端显示)
**后端 my-tasks ChangeRequest 已有**: output(总结)、source_id(应用ID)、created_at、reviewer。前端 Chg type 补全这些字段即可。

**前端显示**(每条待上线):
```
✅ UI 样式升级和布局优化          [上线]
   app: snake · 提交: admin · 07-18 00:30
```

### 3.2 应用部署上线(关联变更)
**当前**: promote 直接部署 HEAD,不关联变更。
**改进**: 应用部署页 prod 区块显示"待上线变更"(approved + source_id=该应用),点「上线」时显示变更摘要确认。

**后端**: detail 已返回 changes(含 approved)。前端应用卡片 prod 区块显示 approved 变更摘要。

## 4. 改动
- **前端 page.tsx**: Chg type 补全字段 + TaskGroup 显示 sub info
- **前端 applications/page.tsx**: prod 区块显示 approved 变更
- 后端无需改(数据已齐)

## 5. 验收
- AC1: 首页待上线显示 功能摘要+应用+提交人+时间
- AC2: 应用部署 prod 区块显示 approved 变更摘要
- AC3: 上线后待上线列表减少
