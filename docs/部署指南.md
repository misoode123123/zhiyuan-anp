# 智源 ANP 部署指南

## 一、环境要求

| 组件 | 版本 | 用途 |
|---|---|---|
| Go | 1.22+ | 平台后端 |
| Node / pnpm | 20+ / 10+ | 前端 |
| Python | 3.10+ | AI 运行时 |
| opencode | 最新 | 编码引擎（研发工作台） |
| 智谱 API Key | — | GLM 模型（需求规格/编码/测试） |

> Windows 环境：需 Git Bash（运行 `scripts/dev.sh`）。

## 二、开发启动（一键）

### 1. 配置 AI Key
编辑 `platform/agent-runtime/.env`：
```env
ZHIPUAI_API_KEY=你的智谱key     # 申请：https://bigmodel.cn
DEFAULT_MODEL=zhipu/glm-4-flash
```

### 2. 安装编码引擎
```bash
curl -fsSL https://opencode.ai/install | bash
opencode --version   # 确认
```

### 3. 一键启动
```bash
bash scripts/dev.sh
# Go 后端    http://localhost:8080
# AI 运行时  http://localhost:8001
# 前端       http://localhost:3000  ← 浏览器打开
```

### 4. 验证
- 健康检查：`curl http://localhost:8080/healthz`
- 前端首页应显示「智源 ANP 平台概览」

## 三、配置说明

| 配置 | 位置 | 说明 |
|---|---|---|
| 基础运行（DB/端口） | `.env` / 环境变量 | 启动前需要 |
| 业务配置（模型/key/opencode） | **系统配置页** `/admin/config` | 入库管理，**热生效** |
| opencode 模型路由 | `platform/opencode.json` | 智谱 provider |

> 业务配置建议从「系统配置」页改，不要散落在文件里（防遗忘）。

## 四、生产部署要点

| 组件 | 建议 |
|---|---|
| Go 后端 | `go build` 单二进制，K8s 多副本 |
| Python AI 运行时 | 独立 Deployment，经 gRPC/HTTP 供后端调用 |
| 前端 | `pnpm build` + CDN/Nginx |
| 数据库 | SQLite → PostgreSQL（装 Docker 后改 `DATABASE_URL`） |
| 算力 | 模型 API（智谱）或自建 GPU 池 |
| 编码引擎 | opencode 常驻 `opencode serve`，平台调度 |

## 五、常见问题

| 问题 | 解决 |
|---|---|
| 前端全 500（Turbopack panic） | 清 node 残留 + 删 `.next`，重启 dev |
| opencode 编码超时 | 已异步：POST 立即返回 task_id，轮询 `GET /code-tasks/:id` |
| 项目空间中文乱码 | 用浏览器「系统配置/侧边栏」创建（UTF-8），勿用 curl |
| 编码被规则阻断 | 「规则治理」里有 block 规则命中，去审批或调整规则 |
| 模型未配置 | 「系统配置」页填 `zhipuai_api_key` |

## 六、首条主线验证（快速验收）

1. 「需求工作台」输业务描述 → 「生成需求规格」
2. 规格卡片点「⚡派发编码」（填仓库路径，如 `D:/.../pilots/oc-pilot`）
3. 「🚪变更审批」→ 找到该变更 → 「批准」
4. 「🚀发布中心」→ 该变更「发布」
5. 回「需求工作台」→ 该需求显示「✅已交付」

走完即验证主线全闭环。
